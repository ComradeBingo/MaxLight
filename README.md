**Max Light** — это обёртка для веб-мессенджера Max (web.max.ru), сочетающая в себе возможности десктопной версии с рядом преимуществ.

## :zap:Основные отличия от официальной версии для Windows ПК

|Свойство| MAX| MaxLight |
|------|------|-----|
|**Вес установщика**| >300 мб|< 10 мб|
|**Обновления**| Полная переустановка (скачать новую версию, удалить старую, заново установить)| Дельта-обновления (только новые файлы). Обновление в фоне за пару секунд|
|**Мониторинг за пользователем**| 8 трекеров слежения | Трекеры заблокированы |
|**Потребление ресурсов**|Высокое. Грузит видеокарту| Не больше, чем у Telegram|
|**Исходный код**|Полностью закрыт|Открытый исходный код. MIT лицензия (кроме самой веб-версии MAX)|
|**Хранение локальных данных**|Да, всё в локальном кэше|Каждый раз при запуске создается временная папка с автоматической очисткой. Нет, ничего "догонять" из старых сообщений и файлов не придётся|


## 🔒 Безопасность

### Шифрование данных
- Все чувствительные данные (токен, deviceId, PIN-код) хранятся в зашифрованном виде в реестре Windows.
- Используется **DPAPI (Data Protection API)** с привязкой к текущему пользователю Windows.
- Алгоритм: AES-256 через Windows Crypto API.

### Защита от XSS
- Внедрение специальных скриптов защиты в WebView2.
- Санитизация ввода и CSP-заголовки.

### Блокировка трекера
Фильтруются HTTP-запросы, содержащие:
- `analytics`
- `apptracer`
- `perf/`, `sdk-api`
- `adsystem`, `crashtoken`
- `crash`, `track`

### PIN-код
- Возможность установить пин-код для программы, если компьтером пользуется кто-то ещё.

## 📥 Установка и обновление

### Установка
1. Скачать `MaxLight-win-Setup.exe` из последнего релиза
2. Запустить установщик
3. Следуйте инструкциям на экране

### Автообновление
При каждом запуске приложение проверяет GitHub на наличие новой версии и автоматически устанавливает её.

### Ручное обновление
Скачать и запустить новый `Setup.exe` поверх старой версии.

⬇️ **Техническая информация не для каждого**⬇️

## 📦 Система обновлений (Velopack)

| Параметр | Значение |
|----------|----------|
| Источник | GitHub Releases |
| Тип обновлений | Дельта (скачиваются только изменённые файлы) |
| Проверка | При каждом запуске приложения |
| Установка | Фоновая, с перезапуском приложения |

### Файлы релиза
- `MaxLight-{version}-win-full.nupkg` — полный пакет
- `MaxLight-{version}-win-delta.nupkg` — дельта-обновление
- `MaxLight-win-Setup.exe` — установщик
- `releases.json` — манифест обновлений

## :wrench: Техническая информация и описание проделанной работы
:one:Внутри оболочки запускается браузер в режиме inprivate (Инкогнито). Это позволяет не хранить кеш локально.Временная папка для работы удаляется аавтоматически после завершения работы. Авторизация требуется только один раз (в отличии от обычной работы в браузере "в режиме инкогнито"). Как? Читаем далее.<br>
:two:При первом запуске приложение предлагает создать пин-код. Если отказаться - можно потом добавить его в настройках. <br>
3️⃣Затем приложение ищет зашифрованный токен авторизации, если его нет - парсер следит за авторизацией до получения токена. После авторизации сохраняет токен в зашифрованом виде. Информацию об авторизации (токен) можно удалить в один клик в настройках.<br>
4️⃣ Система уведомлений целиком и польностью написана с нуля. Парсер следит за чатом определяя два вида бейджей непрочитанных сообщений. Игнорирует серые (уведомления чата отключены пользователем), реагирует на синие (новое непрочитанное сообщение). Парсит имя\ник , аватарку, тело сообщения. На основании этих данных создает уведомление - всплывающее окно в трее. Уведомление не сворачаивает 3D приложения. Можно закрыть нажав на крестик. Клик по уведомлению открывает чат с собеседником. Если вы пропустили уведомление, то: а) при свернутом приложении в панели задач иконка приложения начинает мигать через 2 секунды, затем просто светится. б) Если приложение свернуто в трей (вы закрыли окно), то иконка в трее приобретает красную точку + указание количества непрочитанных сообщений при наведении курсора на трей. При поступлении звонка через MAX окно автоматически разворачивается. <br>
5️⃣ Парсер написан на Javascript и зашит в .exe файл, чтобы избежать внесения изменений злоумышленниками. Целостность файлов (SHA1) при обновлении контролируется Velopack. XSS защита в коде приложения. Всё это несколько увеличиывает вес программы и потребляемые ресурсы (допустимо сделать сборку из одного .exe и нативной библиотеки, общим весом в 2,5мб и потреблением всего 15 мб RAM). Про список заблокированных трекеров уже писалось выше.

**P.S.:** К сожалению, данная программа по сути является лишь многофункциональной надстройкой и не решает большинство проблем MAX, поэтому сквозное шифрование пока что остается на совести официальных разработчиков.

## ⚙️ Конфигурация и хранение данных

| Данные | Место хранения | Шифрование |
|--------|----------------|-------------|
| Токен авторизации | `HKCU\SOFTWARE\MaxLight\Auth` | ✅ DPAPI |
| DeviceId | Внутри зашифрованного AuthData | ✅ DPAPI |
| PIN-код | `HKCU\SOFTWARE\MaxLight\PIN` | ✅ DPAPI |
| Размер/позиция окна | `HKCU\SOFTWARE\MaxLight\WindowState` | ❌ Нет |
| Автозапуск | `HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run` | ❌ Нет |
